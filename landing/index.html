Why it exists

Unused skills aren't free.

Every skill your agent loads is text it has to read past — and pay for — on every single turn.

▲Context rot

Frontier models lose 30%+ accuracy at mid-window positions long before the window is "full." Noise in the context actively degrades answers.

▲The discovery ceiling

Past ~32 installed skills, Claude Code truncates even the skill descriptions. Native lazy-loading was requested — and closed "not planned."

▲Libraries keep growing

Superpowers, GSD, marketplaces — skill counts only go up. Every install makes every turn heavier, unless something prunes intelligently.

What you get

A scalpel, not a switch.

●Fully local

BM25 ranking + trigger rules. Runs in milliseconds, no GPU, no model download. Your prompts and code never leave the machine.

●Zero paid calls

Other skill routers bill you Haiku tokens per prompt. slimcontext costs nothing to run — ever.

●Savings dashboard

slimcontext stats shows cumulative tokens saved. No other skill tool ships this.

●Reversible

Skills are moved, never deleted. slimcontext restore puts the full library back, always.

●Automatic mode

A UserPromptSubmit hook scores and advises on every prompt. It can never block or break a session.

●Tunable

Drop a slimcontext.yaml into any skill: keywords, globs, extensions, alwaysLoad, dependencies.

Quick start

Lean in three commands.

1

Install

npm install -g slimcontext — Node 18+, no native modules.

2

Stage your skills for the task ahead

slimcontext apply "refactor the auth module" — parks irrelevant skills so your next session starts lean. Undo anytime with slimcontext restore.

3

Or let it run itself

slimcontext init installs the hook. Then watch the wins pile up with slimcontext stats.

What slimcontext does not do: it never touches MCP tool loading — Claude Code shipped native "MCP Tool Search" in January 2026. slimcontext is about your skills, and nothing else does this for them.
